# Security Policy

## Supported Versions

Security updates are provided for the latest version of this project on the default branch.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by contacting:

- Developer: Bornil Mahmud
- Email: bornilprof@gmail.com

Please include the following details in your report:

1. A clear description of the vulnerability.
2. Steps to reproduce the issue.
3. The potential impact.
4. Any suggested remediation, if available.

## Response Commitment

- Initial acknowledgment target: within 3 business days.
- Triage and validation target: within 7 business days.
- Remediation timeline depends on severity and complexity.

## Disclosure Policy

Please do not publicly disclose vulnerabilities until a fix or mitigation has been prepared and released.
